Pipeline de Entrenamiento Actual

1. Revisar desarrollo previo en AITuring

En esta etapa se determina si existe un proyecto similar en los repositorios de AITuring

  • codecommit

  • bitbucket

  • github

De encontrarse un proyecto similar, se procede a realizar pruebas con ese desarrollo previo. Se modifica de acuerdo a las nuevas especificaciones de diseño y posteriormente, se presenta resultados.

2. Investigar enfoques similares en base de datos, conferencias, papers

El estado del arte consta de artículos científicos en bases de datos, publicaciones en conferencias, talleres, recursos en la internet, libros, y expertos en su campo.

3. Realizar pruebas en el estado del arte

Una vez se ha encontrado un recurso/proyecto similar, se inicia la etapa de prueba y puesta en marcha para determinar la funcionalidad del mismo. Esta etapa consta de refactorización, adecuación e integración del estado del arte en el nuevo pipeline.

4. Proponer un pipeline para el nuevo desarrollo

Es en esta etapa en la que se propone una nueva solución teniendo como referencia base un desarrollo del estado del arte o en su defecto, un desarrollo previo de AITuring.

  • Primero se realiza una discusión grupal sobre las etapas o fases del nuevo pipeline.

  • Se determina el alcance del proyecto

  • Se determina las posibles limitaciones del proyecto

  • Se determinan los objetivos generales y especificos

  • Se determina el tiempo aproximado de entregas

  • Se asignan tareas con base en los objetivos planteados

  • Se discuten procedimientos de experimentación: programas, tools, repositorios, practicas de desarrollo en equipo, standards, etc

5. Realizar pruebas con el nuevo pipeline

Etapa de experimentación que consta de puesta en marcha del nuevo pipeline, dado unos objetivos definidos en la etapa anterior. Generalmente si existe un desarrollo previo en una fuente de desarrollo de software libre, se descarga el repositorio en local para continuar y/o modificar el proyecto de acuerdo a los objetivos planteados. También, se determina si se necesitan recursos de GPU, CPU especificos que requieran una máquina virtual. Adicionalmente, se crea un entorno de trabajo independiente para cada proyecto en donde se instalan todas las dependencias. Esto permite tener un espacio de trabajo especializado unicamente a ese desarrollo en concreto que se está llevando a cabo.

  • Etapa de experimentación para Machine Learning

De encontrarse o proponerse un desarrollo con fundamentos en Machine Learning, se define un ciclo de vida del proyecto.

5.1. Este ciclo es iterativo en dónde el primer paso es definir el alcance y la finalidad del proyecto; si se necesita un detector de objetos, identificar un producto, conocer áreas de interés, clasificar zonas, entre otros. Este punto orienta el desarrollo del proyecto.

5.2. Luego, una vez se tiene definido el alcance del proyecto se procede a establecer el tipo de dato que se necesita obtener.

Las personas entienden mejor tipos de datos sin estructura cómo imagenes, audio y texto. En contraste, los algoritmos entienden mejor tipos de datos estructurados. Con base en esto, se propone un pipeline de adquisición de datos basado en el tipo (imagenes en el caso de AITuring), se propone un baseline (lo que es posible alcanzar, usualmente es el Human Level Performance HLP) y se capacita al cliente en el uso del aplicativo para captura de fotos (Procedimiento Cliente Nuevo).

5.3. Labeling es la etapa de marcación de imagenes. Existe un equipo dedicado a esta tarea en AITuring y son quienes crean los conjuntos de datos (datasets) utilizados para la creación de modelos de Machine Learning. Las buenas practicas de un proyecto de Machine Learning establecen tips y convenciones durante una etapa de labeling. Es importante la consistencia durante esta etapa de marcación, pues un labeler puede marcar de 3 distintas maneras (ver imagen).

Si el proyecto de vida de ML es de tipo model centric (el que usa actualmente AITuring), solo se marcan las imagenes, se crea un dataset y posteriormente se empieza la etapa de entrenamiento del modelo. En caso de que el proyecto de vida de ML sea data centric, el core del proyecto se encuentra en la etapa de labeling, pues el ciclo de mejora continua se centra en el dataset y no en los parametros del modelo.

5.4. Modeling es la etapa en la que se crea el pipeline de entrenamiento. Es una red neuronal que conecta multiples capas para detección de puntos claves en las imágenes y permite generar un algoritmo de inferencia. En este proceso se crea un modelo que predice el tipo de dato de entrada (ver imagen: Pipeline de Entrenamiento Actual al inicio de este documento).

5.5. Con el modelo entrenado, se procede a validar su performance utilizando herramientas de analisis como matriz de confusion, accuracy, f1 score, entre otros. Esta etapa es muy importante, pues define el gap de mejora del modelo con respecto al baseline definido en 5.2.

5.6. Deployment es el proceso de desplegar un modelo entrenado a un servidor. En el caso de AITuring, se comprime el modelo entrenado en formato .tar.gz y se guarda en un bucket de AWS S3, donde posteriormente es usado por el equipo de operaciones.

5.7. Monitoreo es la etapa final y se caracteriza por una supervisión del modelo en funcionamiento real con el cliente. Actualmente no se realiza esta tarea en el proceso operativo de AITuring (o no se conoce en la documentación proporcionada al equipo de inteligencia artificial).

6. Comparar y evaluar el pipeline con propuestas similares en el estado del arte

Una vez se ha alcanzado la etapa de resultados, se realizan evaluaciones del perfomance del nuevo pipeline en comparación de las propuestas en el estado del arte. Esto permite determinar el grado de mejora que puede tener el proyecto, el base line del mismo y los posibles desarrollos futuros.

7. Documentar el nuevo pipeline

Etapa final en la que se documenta todo el proceso de desarrollo del nuevo pipeline. Esta etapa consta de documentación en Confluence, documentación en el repositorio desarrollado, documentación de diagrama de bloques de sistemas, documentación de referencias, documentación de resultados, documentación de recursos, entre otros.
