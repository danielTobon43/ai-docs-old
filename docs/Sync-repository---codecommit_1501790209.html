Inteligencia Artificial - documentación central : Sync repository - codecommit

Created by Daniel Tobon, last modified on Nov 05, 2021

Es posible que ya esté utilizando Atlassian Bitbucket para el almacenamiento de repositorios, pero tal vez desee descargar su Integración continua a AWS por una variedad de razones. Ver más AWS Codecommit push from Bitbucket pipeline | medium.

Suponemos aquí que tiene un repositorio de Bitbucket existente que desea sincronizar con AWS CodeCommit.

1. Configure Bitbucket Pipelines

Para usar Bitbucket Pipelines, primero debe habilitarse para el repositorio. En la configuración de su repositorio (Repository Settings), elija Settings en el tab “PIPELINES” y habilite los pipelines.

2. Ssh Keys

Lo primero que todo, es necesario crear claves ssh en la sección pipelines

IMPORTANTE
No utilice “Use my own keys“, en su lugar, de click en “Generate keys“

Bitbucket mantiene la clave privada ssh en /opt/atlassian/pipelines/agent/ssh/id_rsa

3. AWS configuration

AWSCodeCommit access es proporcionado por Nelson. Este paso es a manera de información.

Setup. Crear un nuevo usuario desde la consola de AWS IAM

Otorgue dos permisos a ese nuevo usuario de codecommit:

AWSCodeCommitFullAccess y
AmazonS3FullAccess

3.1 Cargue la clave la llave pública ssh que generó en Bitbucket (Repository PIPELINES SECTION) en la sección de credenciales de usuario de AWS (aws user credential).

  • Vaya a la sección de claves ssh de Bitbucket y copie su clave pública.

  • Regrese a la consola de AWS y haga clic en "Upload SSH Public Key" en la sección “SSH Keys for AWS CodeCommit“. Pegar, aceptar.

Ahora, copie el SSH key ID.

  • APK…

4. Create config file

nano ~/.ssh/configBitbucket

Tenga en cuenta que debe pegar el SSH key ID generado en la consola de AWS IAM, aquí

Host git-codecommit.*.amazonaws.com
User YOUR_USER_SSH_KEY_ID_HERE
IdentityFile /opt/atlassian/pipelines/agent/ssh/id_rsa

5. Environment Variables

Configuremos las variables de entorno de Bitbucket para el repositorio. En la configuración de su repositorio, elija “Repository Variables” en PIPELINES.

Vamos a crear 4 variables de entorno como se muestra a continuación.

Aquí está a alto nivel de lo que son, y los cubriremos con más detalle a continuación:

  • CodeCommitConfig: la versión codificada en base64 de la configuración SSH que agregamos a nuestro ~ /.ssh/configBitbucket anteriormente que especifica el Host, el Usuario y el IdentityFile.

  • CodeCommitHost: el host y la región de su instancia de CodeCommit

  • CodeCommitRepo: el host, la región y la ruta del repositorio de su repositorio.

  • CodeCommitUser: el ID de clave SSH asociado con la clave pública en su usuario de AWS IAM.

codificar a base64, ejecutar:

base64 ~/.ssh/configBitbucket

También, puede codificar en base64 un archivo aquí o un texto aquí.

La variable CodeCommitConfig está codificada en base64 para asegurarse de que todos los caracteres estén codificados de manera que podamos referirnos a ellos más adelante. Dado que cada variable es solo una línea de texto, las variables de varias líneas no se mantendrán correctamente si intenta pegarlas sin codificarlas primero.

5. Add Bitbucket pipeline to project

Ahora, crearemos el archivo bitbucket-pipelines.yml, agréguelo con su editor favorito a la raíz de su proyecto (repositorio) o haga clic en "Configurar bitbucket-pipelines.yml" y edítelo directamente en http://bitbucket.org . 

pipelines:
  default:
    - step:
        script:
          - echo $CodeCommitConfig > ~/.ssh/config.tmp
          - base64 -d  ~/.ssh/config.tmp > ~/.ssh/config
          - cat ~/.ssh/config
          - set +e
          - ssh -o StrictHostKeyChecking=no $CodeCommitHost
          - set -e
          - git remote add codecommit ssh://$CodeCommitRepo
          - git push codecommit $BITBUCKET_BRANCH

Ahora, cada vez que ejecute “git push“ a bitbucket, también se actualizará el repositorio en codecommit.
