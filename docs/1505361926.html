Descripción

En procesamiento digital de imagenes un punto de interés o feature son patrones específicos o características específicas que sean únicas, que se puedan rastrear fácilmente, que se puedan comparar fácilmente.

Tomado de Feature extraction and similar image search with OpenCV for newbies. medium

Pipeline

  1. Buscamos estas características en una imagen, las encontramos,

  2. Encontramos las mismas características en otras imágenes,

  3. Las alineamos

El proceso de extracción de caracteristicas consiste en identificar regiones de interes que puedan ser en común con multiples imagenes.

Approach para feature detection and matching

Para este prueba de concepto se ha propuesta la implementación de multiples algoritmos de detección, extracción y alineación de caracteristicas locales. Esto permite identificar objetos en imagenes con caracteristicas similares a un template.

Algunos de los algoritmos más populares para detección y extracción de caracteristicas son:

  • SIFT(Scale Invariant Feature Transform)

  • SURF(Speeded Up Robust Feature)

  • BRISK (Binary Robust Invariant Scalable Keypoints)

  • BRIEF (Binary Robust Independent Elementary Features)

  • ORB(Oriented FAST and Rotated BRIEF)

Tomado de Introduction To Feature Detection And Matching. medium

Idealmente, esta información sería invariable bajo la transformación de la imagen, por lo que podemos encontrar la característica nuevamente incluso si la imagen se transforma de alguna manera. Después de detectar el punto de interés, calculamos un descriptor para cada uno de ellos.

Un descriptor de características es un algoritmo que toma una imagen y genera vectores de características. Los descriptores de características codifican información interesante en una serie de números y actúan como una especie de "huella digital" numérica que se puede utilizar para diferenciar una característica de otra.

Luego que se ha computado los descriptors de cada imagen, se procede al cálculo de la alineación de caracteristicas o “matches“ entre un par de imagenes. La coincidencia de características o la coincidencia de imágenes en general, es la tarea de establecer correspondencias entre dos imágenes de la misma escena/objeto. Una vez que las características y sus descriptores se han extraído de dos o más imágenes, el siguiente paso es establecer algunas coincidencias preliminares de características entre estas imágenes.

El procedimiento de matching es el siguiente:

  • Encuentra un conjunto de puntos clave distintivos

  • Definir una región alrededor de cada punto clave

  • Extraer y normalizar el contenido de la región

  • Calcule un descriptor local de la región normalizada

  • Alinear con descriptores locales

Algunos algoritmos de correspondencias son:

  • Brute-Force Matcher

  • FLANN(Fast Library for Approximate Nearest Neighbors) Matcher

Approach para matching con multiple instancias en una imagen

El problema con el enfoque de alinear correspondencias entre un par de imagenes de la misma escena es que solo pudo detectar una instancia de la plantilla en la imagen de entrada; ¡no podría realizar la detección de múltiples objetos!

Existen varios enfoques para lograr esto la alineación de multiples objetos en una imagen con un template:

  • CNN

  • Viola and Jones detector

  • Haar cascade

  • Template matching

  • Clustering

En el caso de alineación de multiples objetos con un template, usualmente el procedimiento es el siguiente:

  • Aplicar el matchTemplate

  • Encuentar todas las coordenadas (x, y) donde la matriz de resultados coincidentes de la plantilla sea mayor que una puntuación de umbral preestablecida

  • Extrae todas estas regiones

  • Aplicarles supresión no máxima

Otro enfoque propuesto fué el uso de un algoritmo de Clustering.

El análisis de clusters o clustering es la tarea de agrupar un conjunto de objetos de tal manera que los objetos del mismo grupo (llamado clusters) sean más similares (en cierto sentido) entre sí que a los de otros grupos (clusters). Es una tarea principal del análisis de datos exploratorios y una técnica común para el análisis de datos estadísticos, que se utiliza en muchos campos, incluido el reconocimiento de patrones, el análisis de imágenes, la recuperación de información, la bioinformática, la compresión de datos, los gráficos por computadora y el aprendizaje automático.

Dado que la detección y extracción de caracteristicas en imagenes muestra en un patrón de agrupación (tal y como se muestra en la figura de abajo), es posible utilizar un algoritmo de clustering para la agrupación de caracteristicas cercanas en subgrupos, y de esta manera, implementar un procedimiento de alineación de caracteristicas entre multiples objetos.

Para esta prueba de concepto, se ha propuesta el uso de algoritmos de clustering para el computo de matching entre grupos de features cercanos. Un ejemplo es como se muestra en la figura de abajo.

Este enfoque tiene ventajas y desventajas en determinadas situaciones.

  • Si se tiene 2 imagenes de la misma escena (por ejemplo 2 afiches similares), entonces los resultados de clásificación son óptimos, pues los puntos de interés globales de cada imagen son semejantes.

  • Si se tiene 2 imagenes con variaciones en la escena (por ejemplo varios afiches en una imagen), se empiezan a observar agrupaciones debido al numero de clusters.

  • Un problema que se presenta es que el enfoque de clusters no puede ser aplicado a 2 imagenes de la misma escena.

Conclusion

Se recomienda el uso de un detector de rectangulos o recuadros, para la clasificacion de multiples afiches en una imagen. El metodo de clustering sería una alternativa siempre que se limite el conjunto de imagenes a multiples afiches. Si se tienen imagenes con 1 solo afiches, el algoritmo de clustering no es adecuado pues sub-divide el conjunto de caracteristicas de interes globales de la imagen.

Hough line transform

Con base en los resultados obtenidos con la propuesta de clustering, se ha decido probar un enfoque basado en detección de lineas utilizando Hough transform, contornos y filtros dado que los afiches o posters son en geometría rectangulares.

Resultados

Se determinó que generalizar un algoritmo para detectar la ubicación de los posters en cada imagen utilizando Hough o contornos no es óptimo para este caso, pues la escena varía entre imagen e imagen lo que dificulta el proceso de parametrización de cada algoritmo.

Contours Resultados

Aquí se presentan los resultados obtenidos utilizando contornos para la detección de posters. La idea fué calcular el contorno de mayor area dado un radio de area que es la relación entre el tamaño de la imagen y el tamaño del contorno. Los resultados fueron óptimos para la detección de afiches en algunos casos, pero no es un enfoque que se recomiende para producción pues es dificil generalizar el funcionamiento del algoritmo para cualquier escenario.

En su lugar, se recomienda proceder con un enfoque de Deep learning para la creación de un modelo de formas rectangulares utilizando segmentación semantica.
