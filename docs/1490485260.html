Inteligencia Artificial - documentación central : Migración VM instance

Created by Daniel Tobon, last modified by Camilo H on Oct 21, 2021

Procedimiento para migrar una instancia VM de un proyecto a otro

En el proyecto antiguo

Estos pasos se pueden hacer a través de gcloud o la interfaz GCP

  1. Detener la máquina

  2. Editar la máquina y desactivar la opción de borrar el disco cuando se borre la instancia

  3. Borrar la instancia (se requiere borrar para poder crear una imagen nueva)

  4. Ir a discos y con el disco de la instancia (paso 2) crear una instantánea (snapshot)

  5. Ir a imágenes y crear una imagen usando el snapshot del paso anterior

En el proyecto nuevo

Estos pasos se deben hacer únicamente a través de gcloud

  1. Ejecutar el comando de creación de instancia para crear la máquina usando la imagen creada (recordar establecer la zona y el proyecto por defecto antes)

  2. Detener la máquina y cambiar el tipo de máquina (opcional: también se puede hacer en el comando del paso 1 usando los argumentos necesarios)

En el proyecto antiguo

Estos pasos se pueden hacer a través de gcloud o la interfaz GCP

  1. Borrar instantánea creada (ya no es necesaria)

  2. Borrar imagen creada (ya no es necesaria)

  3. Borrar el disco original

Referencia y comandos: https://cloud.google.com/compute/docs/instances/copy-vm-between-projects
