./google-cloud-sdk/install.sh

1.3. Iniciar gcloud init. Ver más gcloud init

./google-cloud-sdk/bin/gcloud init

2. CONFIGURACIÓN SSH KEY

El siguiente paso es configurar la llave de acceso en el metadata del proyecto. Esto se hace mediante llaves ssh que permiten la conexión a las máquinas de manera remota. Para mayor información sobre SSH keys ver la sección Creating a new SSH key

2.1. Crear SSH key

ssh-keygen -t rsa -f ~/.ssh/[KEY_FILENAME] -C [USERNAME]

Reemplazar lo siguiente:

[KEY_FILENAME]: El nombre que desea usar para sus archivos de clave SSH
[NOMBRE DE USUARIO]: El nombre del usuario que se conecta a la VM.

Para gcloud se debe utilizar: google_compute_engine en [KEY_FILENAME].

Para el nombre de usuario puede ser, cloudysanfrancisco@gmail.com o cloudysanfrancisco o cualquier nombre que identifique al desarrollador en la máquina. Tener en cuenta que este nombre de usuario debe utilizarse siempre que se desee interactuar con la máquina a través de gcloud. De lo contrario, se generaría un nuevo usuario en el metadata del proyecto con el nombre del host.

Ejemplo:

ssh-keygen -t rsa -f ~/.ssh/google_compute_engine -C elmorocho

2.2. Restrinja el acceso a su clave privada para que solo usted pueda leerla y nadie pueda escribir en ella.

chmod 400 ~/.ssh/[KEY_FILENAME]

3. LOCALIZACIÓN DE LA NUEVA SSH KEY

Hay varias razones por las que es posible que deba ubicar una clave SSH. Por ejemplo, para agregar la clave SSH pública de un usuario a un proyecto o instancia, necesita acceder al archivo de clave pública para su clave.

Cuando se crea una clave SSH, se guarda en una ubicación predeterminada. Las ubicaciones y los nombres predeterminados de sus archivos de claves SSH públicas y privadas dependen de las herramientas que se utilizaron para crear esa clave. Ver más información en Locating an SSH key.

Si creó una clave en una estación de trabajo Linux o macOS mediante la herramienta ssh-keygen, su clave se guardó en las siguientes ubicaciones:

Archivo de clave pública: ~/.ssh/[KEY_FILENAME].pub
Archivo de clave privada: ~/.ssh/[KEY_FILENAME]

donde [KEY_FILENAME] es el nombre de archivo de la clave SSH, que se estableció cuando se creó la clave.

4. AGREGAR NUEVA SSH KEY AL PROYECTO DE GCP

El acceso de los usuarios a una instancia de Linux a través de herramientas de terceros está determinado por las claves SSH públicas disponibles para la instancia. Puede controlar las claves SSH públicas que están disponibles para una instancia de Linux editando los metadatos, que es donde se almacenan sus claves SSH públicas y la información relacionada.

Se debe dirigir al menú principal en GCP (1), luego seleccionar Compute Engine (2), Metadata (3).

Luego seleccione la pestaña SSH keys.

4.1. Agregar o quitar claves SSH públicas del proyecto en GCP

Para agregar o eliminar claves SSH públicas de todo el proyecto de la Cloud Console :

  1. Ir a Metadata opción en su proyecto.

    Go to Metadata

  2. Sobre SSH Keys, click Edit.

  3. Modifique el project-wide public SSH keys:

    • Para añadir una clave pública SSH key, click Add item.

    • Copie y pegue el contenido de su clave pública google_cloud_compute.pub localizada en ~/.ssh/google_compute_engine.pub en la caja de texto.

  4. Cuando finalice la edición de la nueva clave SSH, click Save.
