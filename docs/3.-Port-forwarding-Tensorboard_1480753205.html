Inteligencia Artificial - documentación central : 3. Port forwarding Tensorboard

Created by Daniel Tobon, last modified on Oct 06, 2021

TensorBoard proporciona la visualización y las herramientas necesarias para la experimentación del aprendizaje automático:

  • Seguimiento y visualización de métricas como pérdida y precisión

  • Visualización del gráfico del modelo (operaciones y capas)

  • Ver histogramas de pesos, sesgos u otros tensores a medida que cambian con el tiempo

  • Proyección de incrustaciones a un espacio dimensional inferior

  • Visualización de imágenes, texto y datos de audio

  • Profiling y mucho más

En esta sección se detalla el proceso de conexión vía ssh con Tensorboard desde una máquina virtual de GCP.

Esto permite la visualización en tiempo real de las métricas de entrenamiento.

  1. Puerto 6006

La conexión de tensorboard a través de gcloud requiere la apertura del puerto 6006.

https://docs.bitnami.com/google/faq/administration/use-firewall/

https://www.google.com/url?sa=t&rct=j&q=&esrc=s&source=web&cd=&cad=rja&uact=8&ved=2ahUKEwjegfKsmrbzAhVRRzABHQTaBjQQwqsBegQIAhAB&url=https%3A%2F%2Fwww.youtube.com%2Fwatch%3Fv%3DsPbBVim9Zbk&usg=AOvVaw0BLMLd2GhGyiUxzqqi9PD8

2. Iniciar tensorboard en máquina

gcloud compute instances start [INSTANCE_NAME]
gcloud compute ssh [USER_NAME]@[INSTANCE_NAME]
user@MY_INSTANCE:~$ tensorboard --logdir LOG_DIRECTORY

3. Abrir conexión ssh

gcloud compute ssh [USER_NAME]@[INSTANCE_NAME] -- -NfL 6006:localhost:6006

4. Abrir tensorboard en navegador web

http://localhost:6006/
