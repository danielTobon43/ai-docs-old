Descripción general de metas para ejecutar la transición a los servicios vertex, sagemaker. Únicamente para la etapa de entrenamiento.

Meta: Efectuar el proceso de entrenamiento en las plataformas vertex, training, tratando de utilizar en la medida de lo posible el código existente de efficient-det

Preámbulo

Existen múltiples maneras de ejecutar los entrenamientos en ambas plataformas (paquete python, container preconfigurado, container personalizado), todas las formas requieren algún tipo de configuración específica dependiendo de la plataforma.

Ventajas

Desventajas

Paquete de python

  • Utilizar un paquete de python, facilita la integración en otros sistemas existentes

  • Permite definir una estructura para la utilización de futuras arquitecturas

  • Requiere esfuerzo de desarrollo del paquete

Container preconstruido

  • No requiere configuración de docker al ser gestionada por el sistema

  • Requiere adaptaciones específicas por plataforma

  • Requiere de paquete de python

Container personalizado

  • Personalización completa de la ejecución de entrenamiento

  • Requiere adaptaciones específicas por plataforma

Todas las alternativas ofrecen ventajas y desventajas que se pueden aprovechar en los distintos entornos de desarrollo manejados por el grupo (local, pruebas, producción), por lo que es posible mantener las 3 formas de ejecución conectadas entre sí: Usar un container personalizado (o preconstruido) que dentro maneje un paquete de python que utilice la arquitectura deseada para el entrenamiento

Objetivos:

  • Ejecutar entrenamiento completo en la plataforma Vertex Training

  • Ejecutar entrenamiento completo en la plataforma Sagemaker

Requerimientos:

  • Creación de paquete de python utilizando cómo base la arquitectura efficiendet

  • Adaptación del docker existente para cumplir los requerimientos de cada plataforma
